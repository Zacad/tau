# Task 043: Google Gemini JSON Schema Compatibility Fix

## Why
Google Gemini API rejects JSON Schema meta fields (`$schema`, `$id`, `$ref`, `$defs`) in tool function declaration parameters. The `github.com/invopop/jsonschema` library generates these fields by default. This causes all tool-calling requests to Gemini to fail with HTTP 400.

## Root Cause
Tau's `GoogleProvider.buildRequest()` passes tool parameters directly from `ToolDefinition.Parameters` (generated by `invopop/jsonschema`) to Google's API. Google's API expects a simplified JSON Schema without meta fields.

## Reference Implementation

### OpenCode (`~/Projects/opencode`)
File: `packages/opencode/src/provider/transform.ts` — `schema()` function
- `sanitizeGemini()` recursively processes schema objects
- Converts integer enums to string enums
- Filters `required` array to only include fields that exist in `properties`
- Removes `properties`/`required` from non-object types
- Ensures `items` has a `type` for arrays

### PI (`~/Projects/pi`)
Same AI SDK approach — the `@ai-sdk/google` provider handles schema normalization internally.

## Constraints
- Must not break other providers (OpenAI, Anthropic, etc.)
- Must preserve valid JSON Schema fields (`type`, `properties`, `items`, `enum`, `required`, etc.)
- Only strip meta fields: `$schema`, `$id`, `$ref`, `$defs`
